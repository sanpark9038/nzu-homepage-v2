# Match Feature Ideas

현재 `/match` 페이지는 "전술 분석실"의 껍데기 UI를 만드는 단계다.
지금은 실제 데이터 연동 없이도 의미 있는 버튼 구조와 자리만 먼저 잡는 것이 좋다.

## 우선 추천 기능

### 1. 상세분석
- 현재 메인 CTA 유지
- 나중에 연결할 내용:
  - 전체 전적
  - 최근 전적
  - 종족전
  - 맵별 전적
  - 최근 흐름

### 2. 일괄 분석
- 여러 매치를 추가한 뒤 한 번에 요약 분석
- 이 페이지 구조와 가장 잘 맞는 기능
- 나중에 연결할 내용:
  - 오늘 등록한 여러 매치업의 위험도/우세도 요약

### 3. 최근전 우선
- 전체 전적보다 최근 3개월 전적을 더 강조하는 모드
- 토글 형태로 두기 좋음

### 4. 소속 필터
- 씨나인, 케이대, 수술대, FA 등 팀 기준 필터
- 현재 파이프라인 메타데이터 구조와 연결하기 좋음

### 5. 변동 배지
- 선수명 옆에 작은 상태칩 표시
- 예:
  - 소속변동
  - 티어변동
  - 신규전적

## UI로 넣기 좋은 버튼/토글

### 상단 또는 중앙 근처
- `전체`
- `최근`
- `가중`

설명:
- 단순 보기 모드 전환 버튼
- 나중에 실제 전적 계산 방식과 연결 가능

### 하단 액션바
- `매치 추가`
- `일괄 분석`
- `초기화`

설명:
- 지금 페이지 흐름과 가장 잘 맞는 조합

## 입력창 주변에 붙이기 좋은 정보

- 팀
- 티어
- 종족
- 최근 변동 여부

예시:
- `씨나인`
- `J`
- `T`
- `소속변동`

## 지금 바로 만들어도 되는 것

- 버튼 자리
- 토글 UI
- 상태칩 디자인
- 일괄 분석 버튼 껍데기

## 지금 만들지 말아야 하는 것

- 실제 전적 계산
- Supabase 연결
- 파이프라인 데이터 실연동
- 실제 상세분석 로직

## 개인적으로 추천하는 우선순위

1. `일괄 분석` 버튼 추가
2. `전체 / 최근 / 가중` 토글 추가
3. 선수명 옆 상태칩 자리 만들기
4. `초기화` 버튼 추가

## 한 줄 결론

`/match`는 단순 검색창보다 "여러 매치를 모아 전술적으로 비교하는 콘솔" 방향으로 키우는 게 가장 좋다.
