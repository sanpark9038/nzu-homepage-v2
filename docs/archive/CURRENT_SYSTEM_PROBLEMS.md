# HOSAGA Current System Problems

This document tracks current technical issues that still matter for the live HOSAGA system.

## Active Problem Areas

- Source-site drift can still break roster or player parsing.
- Metadata hygiene requires periodic review after source changes.
- Pipeline health depends on report validation, not collection alone.
- Supabase sync must remain explicit and verified.

## Non-Problems

- Legacy team-specific branding and project-specific documents are no longer part of the active system.
- Removed teams without active project metadata should not be reintroduced through manual defaults.
