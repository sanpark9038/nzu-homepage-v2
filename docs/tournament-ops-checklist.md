# NZU 홈페이지 대회 운영 체크리스트

대회 준비 단계부터 본 경기, 사후 처리까지 운영자가 어드민 시스템을 이용해 조치해야 할 실무 오퍼레이션 체크리스트입니다. 본 가이드는 별도의 코드 수정 없이 어드민 내 기능만으로 완수할 수 있도록 작성되었습니다.

## 1. 대회 전 (Pre-Tournament)
- [ ] **참가팀 및 로스터 설정 (`/admin/roster`)**
  - 대진에 참여하는 선수들의 출전(수집 예외 여부) 및 티어를 검토하고 최신화합니다.
  - 새 대회의 팀 포맷에 맞춰 **대회 팀명**을 저장하고 반영 여부를 확인합니다.
- [ ] **승부예측 및 일정 등록 (`/admin/prediction`)**
  - 공개된 공식 일정을 바탕으로 팀원 대진, 경기 날짜 및 시작 시각(KST)을 입력합니다.
  - (경기 시간 기준 30분 전 투표가 자동 마감되므로, 시간 설정 시 착오가 없도록 재확인합니다.)
- [ ] **데이터 정상화 모니터링 (`/admin/ops`)**
  - 랭킹 데이터 및 파이프라인의 사전 수집 데이터(선수 전적 등)가 웹사이트 각 페이지에 문제없이 반영되고 있는지 확인합니다.

## 2. 대회 중 (During Tournament)
- [ ] **스케줄 및 대진 변동 대응 (`/admin/prediction`)**
  - 현장 운영 상황에 따라 경기 시간이 지연되거나 일정이 변동될 경우, 투표 조기 마감을 막기 위해 설정된 경기 시간을 즉시 연장/수정합니다.
- [ ] **장애 점검 (`/admin/ops`)**
  - 급작스러운 접속 증가나 UI 오작동, 데이터 미갱신 현상이 없는지 모니터링합니다.

## 3. 대회 후 경기 종료 시 (Post-Match)
- [ ] **경기 기록 승패 갱신 (`/admin/rankings`)**
  - 완료된 경기의 결과를 바탕으로, 해당 날짜에 승/패를 기록한 **팀**과 **출전 선수**의 승수/패수 데이터를 수동으로 즉시 갱신합니다.
- [ ] **다음 대진 예측 점검 (`/admin/prediction`)**
  - 다음 경기가 승부예측 및 일정 페이지에 정상 노출되며 투표 집계가 시작될 준비가 되었는지 체크합니다.
- [ ] **최종 결과 저장 (Optional)**
  - 모든 대회가 종료되어 최종 결과가 확정된 경우, 어드민 승패 입력의 누락이 없는지 최종 교차 검증합니다.
