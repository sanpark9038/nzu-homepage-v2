# Design

This is the short design entry point for the repo's agent-readable operating model.

Read this when the question is "how should the repo be structured so an agent can work safely?"

## Design Principles

- Prefer a map over a manual.
- Keep one concern per file when possible.
- Make invisible decisions visible in markdown.
- Let docs point to the source of truth instead of duplicating it.
