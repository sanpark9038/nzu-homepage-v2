# NZU 성능 및 최적화 전략 가이드 (Performance Strategy)

본 문서는 NZU 전적 플랫폼이 Eloboard 데이터를 기반으로 대규모 데이터를 빠르고 효율적으로 서비스하기 위한 기술적 지침을 정의합니다.

---

## 🚀 4대 핵심 전략 (Core Strategies)

### 1. 사전 수집 및 인덱싱 (Pre-collection & Indexing)
- **원칙**: 모든 데이터는 실시간 크롤링이 아닌, **사전 수집(Batch/Background Sync)**을 원칙으로 합니다.
- **이유**: 실시간 크롤링은 원본 사이트의 지연 시간에 앱의 성능이 종속되지만, 사전 수집은 DB에 최적화된 데이터를 즉시 호출하여 0.1~0.5초 내의 빠른 응답 속도를 보장합니다.

### 2. 가벼운 데이터 모델링 (Lightweight Data Model)
- **축약된 필드명**: JSON 및 DB 필드명을 축약하여 전송량 및 메모리 점유율을 20~30% 절감합니다. (예: `dt`, `rc`, `res` 등)
- **데이터 자료형 최적화**: 
  - 승/패 결과는 문자열("승리")이 아닌 정수형(1/0) 또는 불리언(true/false)으로 저장하여 합산(`SUM`) 연산 속도를 극대화합니다.
  - 상대방 닉네임과 종족 코드(T/P/Z)를 엄격히 분리하여 정규화합니다.

### 3. 미리 계산된 통계 지표 (Pre-computed Metrics)
- **전략**: 사용자가 조회할 때마다 계산하는 것이 아닌, 새로운 데이터가 유입될 때마다 핵심 지표(최근 30일 승률, 상위 맵 승률 등)를 미리 계산하여 저장합니다.
- **성과**: 복잡한 집계 연산 없이 이미 계산된 단일 수치만을 응답하므로 서버 부하를 최소화합니다.

### 4. 데이터베이스 인덱싱 설계 (Database Indexing)
- **필수 인덱스**: `dt(날짜)`, `opp(상대명)`, `rc(상대종족)` 필드에 대해 강력한 DB 인덱스를 구축합니다.
- **목적**: 수만 건의 매치 레코드 중에서도 특정 기간이나 특정 상대와의 H2H(상대 전적)를 지연 없이 즉각적으로 검색하기 위함입니다.
