# MAIN AI UI Redesign Brief (2026-03-23)

이 문서는 메인 AI가 NZU 홈페이지 UI/UX를 다음 단계로 다듬을 때 따라야 할 기준이다.

## 목적

NZU 사이트는 현재 시각적 완성도는 나쁘지 않지만, 사용자 의도는 아래에 더 가깝다.

- 웹/앱 반응형
- 사이트가 무겁지 않을 것
- 검색/필터/핵심 데이터가 빨리 보일 것
- 전체 인상은 깔끔한 미니멀리즘
- 멋진 효과는 허용하지만, 성능과 정보 전달을 해치지 않는 선에서만

핵심 방향은 다음 한 줄로 요약한다.

`지금 NZU의 고급스러운 분위기는 살리되, 정보 구조와 체감 속도는 SSUSTAR + Plausible + StatMuse처럼 더 가볍게 만든다.`

## 현재 NZU UI에 대한 판단

현재 UI는 "못생겼다"가 아니라 "조금 과하다"가 정확하다.

좋은 점:
- 브랜드 톤이 분명함
- 게임/전술 데이터 사이트 느낌이 있음
- 디테일과 시각 완성도는 SSUSTAR보다 높음

문제점:
- blur, glow, shadow, scan line, pulse, hover effect가 한 화면에 너무 많이 겹침
- 히어로 연출이 데이터보다 앞에 옴
- 카드가 많을 때도 카드 1장당 시각 효과가 무거움
- uppercase, tracking, 장식 라벨이 많아 읽는 속도가 떨어짐
- 미니멀리즘보다는 "프리미엄 게임 쇼케이스" 쪽에 가까움

## 절대 원칙

1. 검색/필터/핵심 지표를 장식보다 먼저 보여라.
2. 같은 화면에 강한 시각 효과를 1~2종 이상 겹치지 마라.
3. blur, shadow, glow, motion은 절반 이하로 줄여라.
4. 카드 리스트는 정보 읽기용이어야지 전시용이 아니어야 한다.
5. 모바일에서 바로 읽히는 밀도를 우선하라.
6. 기능 회귀 없이 UI만 더 가볍고 정돈되게 만들어라.

## 가독성 원칙

이 프로젝트에서 미니멀리즘은 "비워 보이게 만드는 것"이 아니라 "읽기 쉽게 정리하는 것"이다.

- 가독성을 최우선으로 할 것
- 폰트를 너무 작게 줄여서 세련돼 보이게 만들지 말 것
- 필요 없는 여백을 일부러 크게 만들어 미니멀한 척하지 말 것
- 정보 밀도는 유지하되 답답하지 않게 정리할 것
- 제목, 표, 카드, 필터, 지표 숫자는 한눈에 읽혀야 할 것
- 시각적으로 조용해도 빈약하거나 힘없는 화면처럼 보이면 안 됨

## 디자인 방향

### 유지할 것

- NZU의 다크 그린 기반 브랜드 톤
- 데이터 사이트다운 진중한 분위기
- 포인트 컬러로서의 그린/골드
- 게임풍 감성의 아주 얇은 흔적

### 줄일 것

- 대형 blur 배경
- 카드별 shadow-2xl 남발
- 반복 pulse / ping / bounce
- scan line, glow, layered FX 중복
- 과한 uppercase / tracking
- 장식용 메타 텍스트

### 강화할 것

- 검색창, 필터바, 상태 요약의 상단 배치
- 정보 블록 간 위계
- 텍스트 대비와 가독성
- 표/리스트/카드의 밀도
- 모바일 세로 레이아웃 품질

## 페이지별 작업 가이드

### 1. Home (`app/page.tsx`)

목표:
- "브랜드 쇼케이스"보다 "바로 쓰는 데이터 홈"으로 전환

수정 방향:
- 대형 히어로 연출 축소
- 좌측 최근 전적 피드와 우측 메인 영역 구조는 유지 가능
- 첫 화면 상단에 검색 또는 빠른 진입 CTA를 더 직접적으로 배치
- 배경 blur, scan line, 장식 메타 텍스트를 크게 줄일 것
- 통계 카운터는 즉시 읽히게 하고, 애니메이션은 최소화하거나 제거

### 2. Players (`app/players/page.tsx`, `components/players/PlayerCard.tsx`, `components/players/Filters.tsx`)

목표:
- "카드 전시"보다 "빠른 탐색" 중심

수정 방향:
- PlayerCard의 시각 효과를 단순화
- 카드 shadow, gradient glow, hover scale를 줄일 것
- 리스트에서 한 번에 더 많은 선수를 읽을 수 있게 정보 밀도 조정
- sticky 필터바는 유지 가능하지만 blur 강도를 낮출 것
- 검색과 종족 필터 사용성이 먼저 느껴져야 함

### 3. Tier (`app/tier/page.tsx`)

목표:
- 데이터 표 느낌을 강화하고, 장식은 보조로만 사용

수정 방향:
- 헤더 장식과 blur 효과 축소
- 상위 티어 구간과 일반 활동 인원의 구분은 유지
- 제목/서브타이틀의 과한 uppercase/tracking 절제
- 표/행의 가독성을 우선

### 4. Player Detail (`app/player/[id]/page.tsx`)

목표:
- "멋진 프로필"보다 "바로 읽히는 선수 대시보드"

수정 방향:
- 대형 blur 배경과 탄소 패턴, 과한 shadow 감소
- 정보 카드 영역을 더 평평하고 읽기 쉽게
- 최근 경기/핵심 지표/소속 정보를 더 직접적으로 보이게 정렬
- 장식 텍스처보다 통계 가독성을 우선

### 5. Entry / Battle Grid / H2H

목표:
- 전술 도구처럼 가볍고 또렷하게

수정 방향:
- 인터랙션은 유지하되 glow/hover 연출 축소
- 패널 간 구조를 더 명확히 분리
- 데이터 비교 영역은 시각 효과보다 표면 구조로 읽히게

## 성능 가드레일

다음은 줄이거나 피할 것:
- 중첩 `backdrop-blur`
- 대형 `blur-[120px+]` 배경 다수
- 반복 `animate-pulse`, `animate-ping`, `animate-bounce`
- 카드 리스트 전체에 과한 shadow/transition
- 필요 이상으로 큰 이미지 hover scale

가능한 방향:
- 섹션당 강조 효과 1개
- 카드 기본 상태는 평평하게, hover는 미세하게
- 애니메이션은 로딩/빈 상태/핵심 CTA 정도에 한정

- 모바일/데스크탑 모두 레이아웃이 자연스러울 것
- 첫 화면에서 검색/핵심 데이터 접근성이 지금보다 좋아질 것
- 시각 효과 수가 체감상 절반 이하로 줄 것
- 카드/상세/테이블의 가독성이 개선될 것
- "화려한데 무거운 사이트"가 아니라 "정제된 데이터 사이트"처럼 느껴질 것
