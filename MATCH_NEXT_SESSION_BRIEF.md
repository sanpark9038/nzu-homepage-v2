# Match Next Session Brief

작성일: 2026-03-24

## 현재 작업 상태

`/match` 페이지는 아직 실제 데이터 연동 전 단계다.
현재 목적은 UI/UX 껍데기를 먼저 완성하는 것이다.

오늘까지 정리된 내용:
- 입력창 UI 정리 완료
- 자동완성 드롭다운 겹침 문제 수정 완료
- 종족 배지 스타일 `T / Z / P` 로 통일
- `P1 / P2` 입력창과 선택창 정렬감 조정
- 중앙 전적 표기를 `전체 / 최근` 기준으로 정리
- `VERSUS` 문구 제거
- 상단에 `전체: 2025.01.01 ~ 현재`, `최근: 최근 3개월 전적` 안내 추가
- 스왑 버튼 위치/가독성 조정
- 상세분석 / 매치추가 버튼 크기와 페이지 밀도 압축
- 전역 배경을 너무 새까만 검정에서 차콜-티얼 계열로 조정

## 이 페이지의 핵심 목적

`/match` 페이지는 경기 직전 빠르게 선수들을 배치하고 밸런스를 맞추기 위한 도구다.

핵심 사용자:
- 방송 스트리머
- 실제 경기 참가 선수

핵심 가치:
- 속도
- 가독성
- 직관성
- 즉각 반응하는 UI

## 반드시 기억할 방향성

이 페이지는 단순한 전적 조회 페이지가 아니다.

정체성:
- 경기 직전용 매치 편성 도구
- 팀 밸런스 확인 도구
- 나중에 방송 송출용 엔트리보드 생성 도구

즉, "전술 분석실"이라기보다 실제 운영용 "매치 메이킹 콘솔"에 가깝다.

## 핵심 기능 요구사항

### 1. 빠른 선수 검색 및 등록
- 자동완성 기반으로 바로 찾을 수 있어야 함
- 느리거나 복잡하면 안 됨

### 2. 선수 등록 즉시 반영될 정보
- 두 선수 간 전체 상대전적
- 두 선수 간 최근 전적

### 3. 팀 밸런스 조정
- 현재 행 단위 스왑은 존재
- 앞으로는 팀 전체 밸런스를 한눈에 볼 수 있는 구조 필요

### 4. 송출용 엔트리보드 생성
- 나중에 버튼 추가 예정
- 버튼 클릭 시 현재 매칭 구성 기준으로 URL 생성
- 선수들이 OBS 브라우저 소스로 추가 가능해야 함
- 참고 링크:
  - `https://overlay-project-one.vercel.app/scoreboard.html`
  - `https://overlay-project-one.vercel.app/entry.html`

## 내일 작업 우선순위

### 1. 레이아웃 구조 재정리
- `/match` 를 "매치 편성 도구" 관점으로 재구성

### 2. 팀 전체 밸런스 요약 영역 설계
- 팀 A / 팀 B 요약
- 평균 티어
- 종족 분포
- 최근 폼
- 밸런스 상태 표시

### 3. 액션 버튼 구조 설계
- `상세분석`
- `일괄 분석`
- `초기화`
- `엔트리보드 생성` (placeholder)

### 4. 나중 데이터 연동을 위한 자리만 확보
- 실제 데이터는 아직 연결하지 않음

## 구현 시 주의사항

- UI 작업만 한다
- 파이프라인 관련 파일은 건드리지 않는다
- `scripts/`, `data/metadata/`, `.github/`, Supabase 쪽은 손대지 않는다
- 빠른 입력과 한눈에 들어오는 구조를 항상 우선한다

## 한 줄 결론

내일 `/match` 작업은 "예쁜 페이지"를 만드는 게 아니라, 경기 직전 빠르게 매치를 짜고 밸런스를 확인하고 나중에 방송 송출까지 이어질 수 있는 운영 도구로 구체화하는 방향으로 이어간다.
